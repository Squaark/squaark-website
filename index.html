The problem

WooCommerce is WordPress
wearing a shop

WooCommerce works. But it's built on top of a CMS from 2003, held together by plugins that conflict with each other, require separate licences, and break on every major update.

You end up maintaining a WordPress site, a WooCommerce install, a payments plugin, a search plugin, a caching plugin — each with its own settings screen, its own update cycle, its own support forum.

squaark is a purpose-built ecommerce platform. Everything it needs is already built in. SQLite is embedded — there's no database server to install, configure, or maintain. One process. You're selling.

Features

Everything you need.
Nothing you don't.

Built from first principles — not bolted together from plugins.

Handlebars themes

Themes are a folder of .hbs files and CSS. No React, no build pipeline required. If you can write HTML, you can build a theme. Upload as a zip and swap live.

SQLite — no database server

SQLite is embedded in the process. No separate database server to install, run, or back up separately. Full-text search, sessions, background jobs — all handled by one embedded file.

One process. That's it.

The entire platform is a single Node.js process. Self-host on any $5/month VPS. No managed Kubernetes, no cloud lock-in, no ops team required. Deploy it like any Node app.

htmx interactivity

Add-to-cart, quantity updates, live theme config preview — all powered by htmx HTML fragments. Pages work without JavaScript. htmx is progressive enhancement, not a requirement.

Built-in product search

SQLite full-text search with relevance ranking and prefix matching. Title matches rank above description matches. No plugin. No external service. No setup. Already there.

WooCommerce import

Migrate your existing store with the built-in import tool. Products, collections, customers, orders — import from WooCommerce CSV or JSON export with a single command.

Live theme editor

Change colours, fonts, and layout from the admin panel. See changes in a live preview iframe before saving. No files to edit, no restart required.

Stripe payments, built in

Checkout, payment intents, order confirmation emails — all included out of the box. No payments plugin to buy, configure, or keep updated. Connect your Stripe account and go.

Secure by default

CSRF on every form. CSP headers on all pages. Argon2id password hashing. Sessions stored securely in SQLite. Theme uploads validated for path traversal before installation.

Self-host

Up and running in minutes,
not hours

One process. No database server. No ops knowledge required.

1

Clone and configure

Clone the repo, copy the example .env file, and set your store name and Stripe keys. That's the whole configuration.

2

Spin up

Run node server.js or docker run squaark/squaark. SQLite is embedded — there's no database server to start. Migrations run automatically.

3

Add products. Start selling.

Log into the admin panel, add your products, configure your theme, connect Stripe — and you're live. Import from WooCommerce if you're migrating.

terminal 
# Clone the repo
git clone https://github.com/squaark/squaark
cd squaark && cp .env.example .env

# SQLite is embedded — no database server needed
# Just start Node and you're running
node server.js

# Or with Docker (single container)
docker run -p 3000:3000 -v ./data:/app/data squaark/squaark

→ Store running at http://localhost:3000
→ Admin panel at http://localhost:3000/admin
