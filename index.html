The problem

WooCommerce is WordPress
wearing a shop

WooCommerce works. But it's built on top of a CMS from 2003, held together by plugins that conflict with each other, require separate licences, and break on every major update.

You end up maintaining a WordPress site, a WooCommerce install, a payments plugin, a search plugin, a caching plugin — each with its own settings screen, its own update cycle, its own support forum.

squaark is a purpose-built ecommerce platform. SQLite is embedded — there's no database server to install or maintain. One process. You're selling.

What's included

Everything built in.
Nothing bolted on.

squaark ships as a single Node.js process. The features below are part of the platform — not plugins, not add-ons, not a marketplace purchase.

Handlebars themes
Themes are a folder of .hbs files and CSS. No React, no build pipeline required. If you can write HTML, you can build a theme. Package as a zip and install from the admin.
SQLite, embedded
No separate database process to install or connect to. SQLite lives inside the Node.js process. Search, sessions, background jobs, config — everything in one file you can back up with cp.
htmx interactivity
Add-to-cart, quantity updates, live theme config preview — all powered by HTML fragments returned from the server. Pages work without JavaScript. htmx is progressive enhancement, not a requirement.
Full-text search
SQLite's FTS5 engine handles product search with relevance ranking and prefix matching. Title matches outrank description matches. No Elasticsearch, no Meilisearch, no Algolia. Already there.
Stripe payments
Checkout, payment intents, order confirmation emails — included out of the box. No payments plugin to license or keep in sync. Connect your Stripe account and go.
Live theme editor
Change colours, fonts, and layout from the admin panel and see changes in a live preview iframe before saving. No files to edit, no restart required.
WooCommerce import
Migrate your existing store with the built-in import tool. Products, collections, customers, and order history — imported from WooCommerce's standard export format.
Secure by default
CSRF protection on every form. Content Security Policy headers on all storefront pages. Argon2id password hashing. Theme uploads validated for path traversal before installation.
Self-hosting

Clone, configure, start.

One Node.js process. SQLite embedded. No database server. Run it on any VPS, any machine, any platform that can run Node.

terminal 
# Clone and configure
git clone https://github.com/squaark/squaark
cd squaark && cp .env.example .env
# Set STORE_NAME, STRIPE_SECRET_KEY, and SESSION_SECRET in .env

# SQLite is embedded — no database server to start or connect to
node server.js

# Or with Docker (single container, no compose file needed)
docker run -p 3000:3000 -v ./data:/app/data squaark/squaark

→  Store running at  http://localhost:3000
→  Admin panel at    http://localhost:3000/admin

Migrating from WooCommerce? The admin panel includes an import tool for products, collections, customers, and order history. Export from WooCommerce, upload in squaark.
